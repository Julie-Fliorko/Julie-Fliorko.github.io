Problem

Weather drives decisions nonstop (training plans, commutes, outdoor events, staffing, travel). But most forecast UX dumps raw variables on users and calls it a day.

This project reframes forecasting into one operational signal: pleasant vs not pleasant, delivered as a probability with drivers ‚Äî so it‚Äôs usable by non-technical stakeholders.

Dataset

Primary Dataset

Historical daily weather pulled via API by location (multi-year horizon).

Typical inputs: temperature, humidity/dew point, wind, precipitation, pressure, cloud cover (varies by source).

Target Engineering

Converted raw variables into a consistent business label: pleasant day (comfort thresholds).

Why it matters: it turns ‚Äúweather data‚Äù into a product metric you can model, monitor, and operationalize.

Approach

1) Data Wrangling + QA Gates

Standardized to daily grain, handled missingness, removed broken rows, and enforced sanity constraints to prevent garbage-in/garbage-out.

2) Feature Engineering (Time-Aware)

Built lag and rolling-window features, seasonality signals, precipitation flags, and short-term volatility metrics ‚Äî designed to generalize, not memorize.

3) Modeling + Tuning

  • Baselines: simple classifiers to set the floor
  • Main: tree-based model (strong accuracy + interpretability)
  • Deep learning: CNN/RNN/LSTM experiments for higher-capacity pattern learning
  • Optimization: grid/random + Bayesian tuning where applicable

4) Evaluation (Decision-Grade)

Compared train/test behavior, reviewed confusion matrices, precision/recall tradeoffs, and probability outputs to make the forecast usable, not just ‚Äúaccurate.‚Äù

Results

  • Signal > noise: engineered time-aware features improved prediction stability versus raw-variable baselines.
  • Seasonality is real: pleasant probability shifts predictably by month/season and location.
  • Top drivers: temperature band + humidity + wind + precipitation (features cluster into ‚Äúcomfort regimes‚Äù).
  • Output is probabilistic: better UX for stakeholder decisions than a brittle yes/no.
